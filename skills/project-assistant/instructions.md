# Skill: Project Assistant for AI-Visual-Style-Assistant

Этот скилл предназначен для помощи в управлении, разработке и публикации проекта AI-Visual-Style-Assistant.

## 🛠 1. Понимание структуры
Проект представляет собой статический веб-сайт.
- `index.html` — основная разметка и структура страницы.
- `style.css` — стили и визуальное оформление.
- `script.js` — логика взаимодействия и функционал.
- `README.md` — документация проекта.

## 🚀 2. Запуск сервиса
Для локального просмотра и тестирования используй один из способов:
- **Быстрый запуск (через сервер):**
  Выполни команду: `npx serve .`
  Затем открой предоставленную ссылку (обычно `http://localhost:3000`).
- **Простой запуск:**
  Просто открой файл `index.html` в любом браузере.

## ✍️ 3. Внесение изменений
При изменении кода следуй этим правилам:
1. **Анализ:** Сначала прочитай содержимое соответствующих файлов (`Read`).
2. **Реализация:** Вноси изменения точечно через `Edit`.
3. **Проверка:** После каждого значимого изменения перезагрузи страницу в браузере и проверь результат.
4. **Стиль:** Придерживайся простого и понятного кода, подходящего для уровня новичка (согласно `CLAUDE.md`).

## 📝 4. Обновление README
Обновляй `README.md` в следующих случаях:
- Добавление нового функционала.
- Изменение способа запуска или установки.
- Обновление списка зависимостей.
- Изменение структуры проекта.
*Следи за тем, чтобы README всегда был актуальным и понятным для нового пользователя.*

## 🐞 5. Проверка ошибок
Если что-то работает не так:
1. **Консоль браузера:** Открой DevTools (F12) -> вкладка Console. Ищи ошибки (красный текст).
2. **Сетевые запросы:** Проверь вкладку Network, если проект загружает внешние данные.
3. **Логи:** Добавляй `console.log()` в `script.js` для отслеживания потока данных.
4. **Синтаксис:** Проверь правильность закрытия тегов в HTML и скобок в JS/CSS.

## 🌐 6. Подготовка к публикации
Чтобы опубликовать проект (например, через GitHub Pages):
1. **Чистка:** Убедись, что в репозитории нет лишних временных файлов или секретов.
2. **Финальный README:** Проверь, что в README есть описание проекта и инструкция по запуску.
3. **Коммит:** Сделай финальный commit всех изменений:
   `git add .`
   `git commit -m "Prepare for publication"`
4. **Пуш:** Отправь изменения в GitHub:
   `git push origin main`
5. **Настройка:** В настройках репозитория GitHub перейди в `Settings` -> `Pages` и выбери ветку `main` для публикации.
